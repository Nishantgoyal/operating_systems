# Managing Local Groups

- Every user on the system has a primary group by default
  - it is named same as the username
  - it is used while creating files
- Then there are secondary or supplementary groups
  - in addition to primary group
  - provides access to files and folder with group permissions
- _groupadd_
  - Syntax: `groupadd <groupname>`
  - `-g`: specify a new ID
- _groupmod_
  - Syntax: `groupmod <options> <groupname>`
  - `-g`: change the ID
- _groupdel_
  - Syntax: `groupdel <groupname>`
- _groups_
  - Syntax: `groups <username>`
  - Lists all the groups the user belong to
  - The first item in the list is the primary group
- To change the primary group of the user:
  - `usermod -g <group_id> <username>`
- To modify the supplementary groups
  - `usermod -G <groups> <username>
  - Will **_replace_** the complete list
  - To append the secondary group, use the `-a` flag
