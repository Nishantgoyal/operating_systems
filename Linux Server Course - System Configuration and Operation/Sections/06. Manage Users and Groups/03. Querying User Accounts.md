# Querying User Accounts

- It is required to do forensics on the system
- Also equally important for doing the sanity check
  - understanding the performance of the system
- There are multiple commands to do the same
  - _whoami_
    - returns the user currently logged in
  - _who_
    - returns all the users currently logged into the system
    - provides the following info
      - name of all users
      - when they logged in
      - where they are logged in from
  - _w_
    - stands for _what_
    - provide more info then _who_
      - how long the user has been idle
      - what command are they currently executing
  - _pinky_
  - _id_
    - provides you the information about any user
  - _last_
    - provides you the list of users that have logged into your system recently
