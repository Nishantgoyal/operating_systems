# Testing DNS

- DNS stands for Domain Name System
- It is the way, the system converts human friendly domain names to IP addresses
- There are many tools which can be used to test DNS
  - _ping_
    - Syntax: `ping <hostname/IP>`
    - sends packets to the host
    - performs DNS resolution if the DNS name of host is provides
  - _dig_
    - Syntax: `dig @server hostname`
    - The `server` (optional) is the DNS server to query for DNS information
    - `hostname` referes to the hostname we are trying to resolve
    - In the response
      - Provides the DNS Server info that was queried
      - Provides `A records` corresponding to the hostname
  - _nslookup_
    - Syntax: `nslookup hostname server`
    - Provides same info as `dig` command
  - _host_
    - Syntax: `host hostname server`
    - Does not provide the DNS Server info in response
    - Provides the `A records` and the `mx records` for the hostname
