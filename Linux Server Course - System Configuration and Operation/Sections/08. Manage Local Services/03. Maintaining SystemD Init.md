# Maintaining SystemD Init

- The SystemD startup modes can be correlated to SysV runlevels

| SysV Runlevel | SystemD Boot Target |
| ------------- | ------------------- |
| 0             | poweroff            |
| 1             | rescue              |
| 2             | multi-user          |
| 5             | graphical           |
| 6             | reboot              |

- To find the current default boot target:
  - `systemctl get-default`
    - will return on of the SystemD Boot Target, like
      - _graphical.target_
      - _multi-user.target_
      - etc
- To change the current default boot target:
  - `systemctl set-default <boot target>`
    - it will not change the current mode in which the operating system is in
    - on next reboot the system will spawn in the specified _boot target_
- To switch between boot targets
  - `systemctl isolate <boot target>`
