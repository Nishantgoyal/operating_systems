# Managing Services with SysV

- Services are the programs installed on a server
- For _SysV_ system,
  - the services are managed by individual files
  - the files for each service is located in _/etc/init.d_ folder
- The `service` command is used to start/stop the services
  - `service <service-name> start/stop/status`
- To see start/stop behaviour of a service on boot
  - `chkconfig --list <service-name>`
  - it will display the behaviour of the service on each run-level
- To start/stop the service on boot on all run-levels
  - `chkconfig <service-name> on/off`
- To start/stop the service on boot on a particular run level
  - `chkconfig --level <runlevel> <service-name> on/off`
